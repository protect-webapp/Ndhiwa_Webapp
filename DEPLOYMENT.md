# Deployment Guide - Ndhiwa Sub-County Children's Office\n\n## Prerequisites\n\n- Node.js v18+\n- npm or yarn\n- Firebase CLI\n- GitHub account\n- Supabase account\n\n## Environment Setup\n\n### 1. Firebase Configuration\n\n```bash\nfirebase login\nfirebase init\n```\n\nSelect:\n- Hosting\n- Functions\n- Firestore\n\n### 2. Set Environment Variables\n\n**Frontend (.env.local)**\n```\nREACT_APP_FIREBASE_API_KEY=xxx\nREACT_APP_FIREBASE_AUTH_DOMAIN=xxx\nREACT_APP_FIREBASE_PROJECT_ID=protect-48d71\nREACT_APP_FIREBASE_STORAGE_BUCKET=xxx\nREACT_APP_FIREBASE_MESSAGING_SENDER_ID=xxx\nREACT_APP_FIREBASE_APP_ID=xxx\nREACT_APP_SUPABASE_URL=xxx\nREACT_APP_SUPABASE_ANON_KEY=xxx\nREACT_APP_API_URL=https://us-central1-protect-48d71.cloudfunctions.net/api\nREACT_APP_ENV=production\n```\n\n**Backend (backend/functions/.env)**\n```\nFIREBASE_PROJECT_ID=protect-48d71\nSUPABASE_URL=xxx\nSUPABASE_KEY=xxx\nCORS_ORIGIN=https://ndhiwa-scco.web.app,http://localhost:3000\nNODE_ENV=production\nENCRYPTION_KEY=your-sha256-encryption-key\n```\n\n### 3. Supabase Setup\n\n1. Create Supabase project at https://supabase.com\n2. Go to SQL Editor\n3. Run migration from `database/migrations/001_initial_schema.sql`\n4. Enable Row Level Security (RLS) on all tables\n\n### 4. GitHub Actions Secrets\n\nAdd to GitHub repository secrets:\n- `FIREBASE_SERVICE_ACCOUNT` - Service account JSON key\n- `ENCRYPTION_KEY` - SHA-256 encryption key\n\n## Local Development\n\n### Start Firebase Emulator\n```bash\nfirebase emulators:start\n```\n\nThis starts:\n- Auth Emulator (port 9099)\n- Functions Emulator (port 5001)\n- Firestore Emulator (port 8080)\n- Hosting Emulator (port 5000)\n- Emulator UI (port 4000) - Visit http://localhost:4000\n\n### Start Frontend (new terminal)\n```bash\ncd frontend\nnpm install\nnpm start\n```\n\nAccess at http://localhost:3000\n\n### Start Backend (new terminal)\n```bash\ncd backend/functions\nnpm install\nnpm run serve\n```\n\n## Production Deployment\n\n### Build and Deploy Frontend\n```bash\ncd frontend\nnpm run build\nfirebase deploy --only hosting\n```\n\n### Deploy Backend Cloud Functions\n```bash\ncd backend/functions\nnpm install\nfirebase deploy --only functions\n```\n\n### Full Deployment\n```bash\n# From root directory\nfirebase deploy\n```\n\n## CI/CD with GitHub Actions\n\nThe `.github/workflows/deploy.yml` automatically deploys to Firebase on push to main branch.\n\n**Setup:**\n1. Create Firebase service account\n2. Add secrets to GitHub repository\n3. Enable GitHub Actions\n\n## Monitoring\n\n### Firebase Console\nhttps://console.firebase.google.com/u/4/project/protect-48d71/overview\n\n### View Cloud Functions Logs\n```bash\nfirebase functions:log\n```\n\n### View Supabase Logs\n- Visit Supabase Dashboard > Logs\n\n## Troubleshooting\n\n### Functions Not Starting\n```bash\n# Check Node version\nnode --version  # Should be 18+\n\n# Clear cache and reinstall\nrm -rf node_modules\nnpm cache clean --force\nnpm install\n```\n\n### CORS Errors\n- Update `CORS_ORIGIN` in `.env`\n- Restart emulator or redeploy functions\n\n### Supabase Connection Issues\n- Verify `SUPABASE_URL` and `SUPABASE_KEY`\n- Check Supabase project is active\n- Test connection in Supabase dashboard\n\n### Firebase Authentication Issues\n- Ensure Firebase project is enabled\n- Check Firebase console for service account permissions\n- Verify `FIREBASE_PROJECT_ID` matches\n\n## Production Checklist\n\n- [ ] Environment variables configured\n- [ ] Firebase Blaze plan enabled\n- [ ] Supabase project created and configured\n- [ ] Database migrations run\n- [ ] SSL certificates configured (automatic with Firebase Hosting)\n- [ ] GitHub Actions secrets added\n- [ ] CI/CD workflow tested\n- [ ] Backup strategy implemented\n- [ ] Monitoring alerts configured\n- [ ] Documentation updated\n\n## Next Steps\n\n1.
